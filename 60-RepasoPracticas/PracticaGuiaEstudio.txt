Guía de estudio: Fundamentos de Ciberseguridad
Cuestionario (10 preguntas de respuesta corta)

    ¿Qué es la virtualización y cuáles son sus componentes principales? La virtualización permite que al menos una computadora virtual funcione dentro de una computadora física. Las computadoras virtuales se conocen como "máquinas virtuales" o "guests", mientras que las computadoras físicas que las albergan se denominan "hosts". Esto aprovecha el aumento de la capacidad computacional y los recursos modernos.
    ¿Cuál es la función principal de Wireshark en un entorno de ciberseguridad? Wireshark es una herramienta de análisis de red utilizada para capturar y examinar el tráfico de red. Permite a los analistas observar los paquetes de datos en detalle, identificar información como puertos de origen y destino, direcciones IP y MAC, y analizar protocolos como UDP y DNS.
    Explica la diferencia entre las marcas de tiempo Unix Epoch y Human Readable en los archivos de registro. Las marcas de tiempo Unix Epoch registran la hora midiendo la cantidad de segundos transcurridos desde el 1 de enero de 1970. Por otro lado, las marcas de tiempo Human Readable registran la hora representando valores separados para el año, el mes, el día, las horas, los minutos y los segundos, haciéndolas más intuitivas para la lectura humana.
    ¿Qué es lsass.exe y dónde se encuentra típicamente en un sistema Windows? lsass.exe es el proceso de autoridad de seguridad local en sistemas Windows. Se encarga de verificar la validez de los inicios de sesión de usuario en la PC. Normalmente se encuentra en la carpeta C:\Windows\System32\. Si se encuentra en otra ubicación, podría ser un indicio de malware.
    ¿Cuál es el propósito del comando journalctl en Linux y cómo se utiliza para ver registros específicos? El comando journalctl se utiliza en Linux para consultar y mostrar entradas del diario del sistema (logs). Puede usarse para ver registros del último arranque (journalctl -b), o para filtrar entradas relacionadas con un servicio específico utilizando la opción -u, como por ejemplo journalctl -u nginx.service.
    Describe el concepto de "ruta absoluta" y "ruta relativa" en el contexto de un sistema de archivos Linux. Una "ruta absoluta" hace referencia a rutas que siempre comienzan desde el directorio raíz (/). Una "ruta relativa" es una ruta más corta que se especifica en relación con el directorio actual. Los directorios . (directorio actual) y .. (directorio superior) se utilizan para trabajar con rutas relativas.
    ¿Por qué se necesita el permiso de root (o sudo) para ejecutar ciertos comandos como ps o ls -l /var/log/nginx en Linux? Algunos comandos, como ps para ver todos los procesos en segundo plano o ls -l en directorios protegidos como /var/log/nginx, acceden a información o recursos que requieren privilegios elevados. Ejecutarlos como root (usando sudo) garantiza que el usuario tenga los permisos necesarios para realizar estas operaciones.
    ¿Cuál es la función principal de la herramienta traceroute (o tracert) y qué información proporciona? traceroute (o tracert en Windows) es una herramienta de diagnóstico de red utilizada para trazar la ruta que toman los paquetes de datos para llegar a un destino. Muestra una lista de los routers ("saltos") que se atraviesan, lo que ayuda a identificar posibles problemas en la red y la conectividad.
    ¿Qué es Nmap y para qué se utiliza comúnmente en seguridad de redes? Nmap es una herramienta de exploración de red y seguridad, también conocida como escáner de puertos. Se utiliza para escanear una red, descubrir hosts activos, identificar puertos abiertos, detectar servicios que se ejecutan en esos puertos y determinar el sistema operativo de los dispositivos.
    Explica la diferencia en el significado del bit de ejecución cuando se aplica a un archivo regular versus un directorio en Linux. Para un archivo regular, el bit de ejecución (x) indica que el archivo es un script o programa ejecutable que el sistema puede correr. Para un directorio, el bit de ejecución (x) especifica si un usuario tiene permiso para entrar o acceder a ese directorio.

Clave de Respuestas del Cuestionario

    ¿Qué es la virtualización y cuáles son sus componentes principales? La virtualización permite que varias computadoras virtuales (guests) se ejecuten en una sola computadora física (host). Este enfoque aprovecha la creciente capacidad de procesamiento y memoria de los sistemas modernos, optimizando el uso de recursos.
    ¿Cuál es la función principal de Wireshark en un entorno de ciberseguridad? Wireshark es una herramienta esencial para la captura y el análisis de tráfico de red. Permite a los profesionales de la ciberseguridad examinar los paquetes de datos en detalle, lo cual es crucial para la resolución de problemas de red, el análisis de vulnerabilidades y la detección de actividades maliciosas.
    Explica la diferencia entre las marcas de tiempo Unix Epoch y Human Readable en los archivos de registro. Unix Epoch registra el tiempo como el número de segundos transcurridos desde el 1 de enero de 1970, siendo un formato numérico ideal para procesamiento de máquinas. Human Readable desglosa la fecha y hora en componentes como año, mes, día, horas, minutos y segundos, facilitando su comprensión por parte de los humanos.
    ¿Qué es lsass.exe y dónde se encuentra típicamente en un sistema Windows? lsass.exe (Local Security Authority Process) es un componente crítico de Windows responsable de la gestión de la seguridad, incluyendo la autenticación de usuarios. Su ubicación estándar es C:\Windows\System32\. Encontrar este ejecutable en una ruta diferente podría ser una señal de una infección de malware.
    ¿Cuál es el propósito del comando journalctl en Linux y cómo se utiliza para ver registros específicos? journalctl es la utilidad principal para consultar el systemd journal, que almacena los mensajes de registro del sistema y de las aplicaciones. Permite filtrar los logs por tiempo (-b para arranques específicos) o por servicio (-u para ver los logs de un demonio particular, como nginx).
    Describe el concepto de "ruta absoluta" y "ruta relativa" en el contexto de un sistema de archivos Linux. Una ruta absoluta especifica la ubicación de un archivo o directorio comenzando desde el directorio raíz (/), proporcionando la ruta completa. Una ruta relativa, en cambio, define la ubicación en relación con el directorio de trabajo actual, utilizando . para el directorio actual y .. para el directorio padre.
    ¿Por qué se necesita el permiso de root (o sudo) para ejecutar ciertos comandos como ps o ls -l /var/log/nginx en Linux? Los comandos que acceden o modifican información sensible del sistema, como la lista completa de procesos en ejecución (ps) o el contenido de directorios de registro críticos (/var/log/nginx), requieren privilegios de root. sudo permite a un usuario ejecutar estos comandos con permisos de superusuario.
    ¿Cuál es la función principal de la herramienta traceroute (o tracert) y qué información proporciona? La herramienta traceroute (tracert en Windows) se utiliza para rastrear la ruta que siguen los paquetes de datos a través de una red IP hasta un destino. Muestra cada router (salto) por el que pasa el paquete y el tiempo que tarda en llegar a cada uno, lo que es útil para diagnosticar latencia y problemas de conectividad.
    ¿Qué es Nmap y para qué se utiliza comúnmente en seguridad de redes? Nmap (Network Mapper) es un escáner de puertos y herramienta de descubrimiento de red ampliamente utilizada. Sus usos incluyen la auditoría de seguridad, la identificación de servicios en ejecución, la detección de sistemas operativos y la creación de inventarios de red para encontrar vulnerabilidades.
    Explica la diferencia en el significado del bit de ejecución cuando se aplica a un archivo regular versus un directorio en Linux. Para un archivo regular, el bit de ejecución (x) permite que el archivo sea ejecutado como un programa o script. Para un directorio, el bit de ejecución (x) otorga la capacidad de "entrar" o "recorrer" el directorio, permitiendo al usuario acceder a su contenido si tiene los permisos de lectura o escritura adecuados.

    Analice cómo la virtualización ha transformado las prácticas de ciberseguridad, discutiendo tanto sus beneficios (por ejemplo, entornos de laboratorio aislados para análisis de malware) como sus desafíos inherentes para la seguridad.
    Compare y contraste las funciones y aplicaciones de Wireshark y Nmap en un escenario de análisis de incidentes de ciberseguridad. ¿Cómo se complementan estas herramientas para proporcionar una visión más completa de una amenaza o vulnerabilidad?
    Discuta la importancia de los archivos de registro del sistema (/var/log/) y la herramienta journalctl en Linux para un analista de seguridad. Incluya ejemplos de tipos de información crítica que se pueden extraer de estos registros y cómo pueden indicar actividades sospechosas.
    Explique la relevancia de los permisos y la propiedad de los archivos en un sistema Linux (chmod, chown) para mantener la seguridad del sistema. Proporcione ejemplos de cómo una configuración incorrecta de permisos podría llevar a una vulnerabilidad de seguridad.
    Considerando el concepto de vulnerabilidad y su explotación, describa el proceso que seguiría un atacante para identificar una vulnerabilidad, quién podría aprovecharla y por qué existiría en primer lugar, utilizando ejemplos generales de las fuentes proporcionadas (sin necesidad de elegir una vulnerabilidad específica).

Glosario de Términos Clave

    Virtualización: Tecnología que permite la creación de una versión virtual de un recurso informático, como un sistema operativo, un servidor, un dispositivo de almacenamiento o un recurso de red.
    Máquina Virtual (VM): Una computadora virtual que se ejecuta dentro de una computadora física. También se conoce como "guest" (invitado).
    Host (Huéspe): La computadora física en la que se ejecutan una o más máquinas virtuales.
    Wireshark: Una herramienta de análisis de protocolo de red (analizador de paquetes) que permite capturar e inspeccionar los datos que viajan a través de una red.
    DNS (Domain Name System): Sistema de nombres de dominio. Un sistema distribuido que traduce nombres de dominio legibles por humanos (por ejemplo, www.example.com) a direcciones IP numéricas.
    UDP (User Datagram Protocol): Protocolo de datagramas de usuario. Un protocolo de comunicación sin conexión utilizado para aplicaciones sensibles al tiempo, como el streaming de video y las consultas DNS.
    Unix Epoch: Un método para rastrear el tiempo, definido como el número de segundos transcurridos desde el 1 de enero de 1970 a las 00:00:00 UTC.
    Human Readable (Legible por humanos): Un formato de marca de tiempo que muestra la fecha y la hora de manera clara y comprensible para las personas (por ejemplo, año, mes, día, horas, minutos, segundos).
    Malware: Software malicioso diseñado para dañar, explotar o controlar un sistema informático sin el conocimiento o consentimiento del usuario.
    lsass.exe (Local Security Authority Subsystem Service): Un proceso en sistemas operativos Windows que gestiona las políticas de seguridad local, la verificación de inicios de sesión y el cambio de contraseñas.
    PowerShell: Un marco de automatización de tareas y configuración de Microsoft que incluye un shell de línea de comandos y un lenguaje de scripting.
    Cmdlets: Comandos ligeros en el entorno de PowerShell, diseñados para realizar funciones específicas. Siguen una estructura de verbo-sustantivo (por ejemplo, Get-Alias).
    netstat: Una utilidad de línea de comandos utilizada para mostrar conexiones de red (entrantes y salientes), tablas de enrutamiento, estadísticas de interfaz, conexiones de pseudo-spoofing y asociaciones de multidifusión.
    ifconfig: Una utilidad de línea de comandos de sistemas operativos tipo Unix utilizada para configurar o mostrar los parámetros de configuración de red de la interfaz de red.
    journalctl: Una utilidad de línea de comandos en Linux para consultar y mostrar mensajes del systemd journal, que recopila los mensajes de registro del sistema.
    Ruta Absoluta: La ruta completa a un archivo o directorio que comienza desde el directorio raíz (/).
    Ruta Relativa: La ruta a un archivo o directorio especificada en relación con el directorio de trabajo actual. Utiliza . para el directorio actual y .. para el directorio padre.
    mkdir: Comando en Linux para crear nuevos directorios (carpetas).
    ls -la: Comando en Linux para listar el contenido de un directorio, incluyendo archivos y directorios ocultos (los que comienzan con un punto).
    Permisos de archivo (Linux): Conjunto de atributos que definen los derechos de acceso de los usuarios y grupos sobre los archivos y directorios en sistemas tipo Unix. Se representan comúnmente en formato octal (por ejemplo, 665, 777).
    chmod: Comando en Linux para cambiar los permisos de acceso de archivos y directorios.
    chown: Comando en Linux para cambiar la propiedad de un archivo o directorio (propietario y/o grupo).
    touch: Comando en Linux para crear archivos vacíos o actualizar la marca de tiempo de un archivo existente.
    Montaje de sistema de archivos: El proceso de hacer que un sistema de archivos en un dispositivo de almacenamiento (como un disco duro o una unidad USB) sea accesible para el sistema operativo en un directorio específico, conocido como "punto de montaje".
    mount: Comando en Linux para montar sistemas de archivos.
    umount: Comando en Linux para desmontar sistemas de archivos.
    lsblk: Comando en Linux que lista información sobre todos los dispositivos de bloques disponibles.
    traceroute / tracert: Herramientas de diagnóstico de red que muestran la ruta y miden los retrasos de los paquetes a través de una red IP hasta un host determinado.
    Nmap: (Network Mapper) Una herramienta de seguridad de código abierto utilizada para descubrir hosts y servicios en una red informática, construyendo así un "mapa" de la red. Se utiliza a menudo para el escaneo de puertos.
    SysInternals: Un conjunto de utilidades gratuitas para monitorear, administrar y diagnosticar sistemas operativos Microsoft Windows. Incluyen herramientas para procesos, monitoreo del sistema, seguridad y redes.
    IDS (Intrusion Detection System): Sistema de Detección de Intrusiones. Un dispositivo o aplicación de software que monitorea una red o sistemas en busca de actividades maliciosas o violaciones de políticas.
    Snort: Un sistema de detección de intrusiones (IDS) de código abierto basado en red, que puede realizar análisis de tráfico en tiempo real y registro de paquetes.
    IPTables: Una utilidad de firewall de línea de comandos en sistemas Linux que permite a los administradores configurar las reglas de filtrado de paquetes del kernel.
    ls -l: Comando de Linux que lista el contenido de un directorio en formato largo, mostrando detalles como permisos, propietario, grupo, tamaño y fecha de modificación.
    ps: Comando en Linux que muestra información sobre los procesos que se están ejecutando actualmente.
    .bashrc: Un archivo de configuración de shell en sistemas Linux que se ejecuta cada vez que se inicia una nueva sesión de terminal interactiva, personalizando el entorno del usuario.
    Kibana: Una plataforma de visualización de datos de código abierto que se utiliza con Elasticsearch para analizar y presentar datos de logs e índices.
